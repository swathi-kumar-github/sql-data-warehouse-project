# SQL Data Warehouse Project

This project demonstrates my ability to design and implement a complete data‑warehouse solution using SQL Server. It follows a structured, industry‑oriented workflow including data ingestion, cleaning, transformation, modeling, and analytical querying.

---

## 🔹 Project Goals
- Build a small end‑to‑end data warehouse using SQL  
- Apply the Medallion Architecture (Bronze → Silver → Gold)  
- Create clean, analysis‑ready tables from raw ERP and CRM datasets  
- Develop SQL queries that generate meaningful business insights  

---

## 🔹 Architecture Overview

### **Bronze Layer**
- Raw ERP and CRM CSV files imported into SQL Server  
- No transformations applied; data stored as‑is  

### **Silver Layer**
- Data cleaning and standardization  
- Handling missing values and formatting inconsistencies  
- Preparing unified tables for modeling  

### **Gold Layer**
- Star‑schema modeling with fact and dimension tables  
- Business‑ready tables optimized for analytical queries  

---

## 🔹 What I Implemented
- SQL scripts for ingestion, cleaning, and transformation  
- Data‑quality checks for Silver and Gold layers  
- Star schema including fact tables (e.g., sales) and dimension tables (e.g., customers, products)  
- Analytical SQL queries to answer business questions such as:  
  - customer behavior  
  - product performance  
  - sales trends
